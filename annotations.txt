// getStaticProps -> faz o 'pre-render' no momento da build
// getServerSideProps -> faz o 'pre-render' a cada requisição

// Vantagens do SSR:

// Não é necessário o método 'getStaticPaths' o qual informa
// ao 'next' quais rotas existem na aplicação, pois ele faz
// isso em tempo real (a cada requisição).

// Não é necessário ter de re-buildar o projeto cada vez
// que uma mudança ocorre na API.

// Uma aplicação 'next' com SSR tem um comportamento parecido
// com algumas tecnologias web conhecidas como Django e PHP
// (buscam os dados da API a cada requisição).

// Usando SSR não é mais possível usar o comando 'npx next export',
// pois a aplicação utilizará não mais a build static e sim a
// build de produção, ou seja, é necessário executar a aplicação
// com o comando 'npm start'.
