// serão BLOQUEADAS, a fim de que os 'buscadores' não

// https://developers-google-com.translate.goog/search/docs/crawling-indexing/robots-meta-tag?_x_tr_sl=en&_x_tr_tl=pt&_x_tr_hl=pt&_x_tr_pto=tc


// É possível usar as metatags também para esse fim.
// -> metatags
// São tags html que permitem os buscadores encontrarem informações
// específicas do website. Eles podem ou não usar essas informações.

// Referências:
// https://yoast.com/meta-descriptions/
// https://developers.google.com/search/docs/crawling-indexing/special-tags


// Open Graph Protocol
// https://ogp.me/
// São valores passados para as metatags que modificam
// a exibição da página quando compartilhada, por exemplo,
// nas redes sociais (aquele card que exibe um resumo da página).

// Por exemplo, o "twitter" possui uma doc sobre as suas metatags, ou seja,
// quando definidas no website criado, ao compartilhar o link do website
// no "twitter", ele procurará por essas metatags e exibirá algumas informações
// no formato de 'card' com detalhes específicos do site.

// -> favicon.ico
// É o ícone referência da aplicação.

// Caso o ícone possua uma extensão de arquivo conhecida:
// <Head>
//     <link rel="shortcut icon" href="favicon.png"></link>
// </Head>

// Se tiver a extensão '.ico', basta colocá-lo na pasta
// 'public' que a aplicação em sí já o reconhece.
